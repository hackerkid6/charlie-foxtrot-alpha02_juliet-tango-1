Basics of Multiple Regression

▶

The Model

Multiple Regression Equation
Yᵢ = b₀ + b₁X₁ᵢ + b₂X₂ᵢ + ... + bₖXₖᵢ + εᵢ

Yᵢ = dependent variable for observation i
X₁ᵢ...Xₖᵢ = independent variables (predictors) for observation i
b₀ = intercept (expected value of Y when all X = 0)
bⱼ = partial slope coefficient on Xⱼ = change in Y per 1-unit ↑ in Xⱼ, all other X held constant
εᵢ = error term / residual for observation i
k = number of independent variables | n = number of observations
Interpreting Coefficients RETRF = 1.53 + 0.589·MKTRF − 0.872·SMB − 0.056·HML
If MKTRF ↑1% (SMB, HML held constant) → portfolio return ↑ 0.589%
Intercept 1.53 = expected excess return when all three factors = 0

Five Regression Assumptions

  1. Linearity — relationship between Y and X's is linear
  2. Homoskedasticity — variance of εᵢ is constant across observations
  3. Independence of errors — residuals are uncorrelated across observations
  4. Normality — residuals are normally distributed
  5. Independence of X's — no exact linear relation between independent variables
Violations of assumptions 2, 3, and 5 are the focus of QM3. All three bias standard errors but NOT the coefficients themselves.

Evaluating Regression Model Fit

▶

Goodness of Fit

R² vs Adjusted R²
R² = explained variation / total variation (always ↑ when variables added → misleading)
Adjusted R² = 1 − [(n−1)/(n−k−1)] × (1−R²)

n = number of observations | k = number of independent variables
Adjusted R² increases only if |t-stat of new variable| > 1; can be negative

Adjusted R² increases only if |t-stat of new variable| > 1
Adjusted R² can be negative; R² cannot
AIC and BIC (Model Selection)
AIC = n·ln(SSE/n) + 2(k+1) → lower is better; prefer for PREDICTION
BIC = n·ln(SSE/n) + ln(n)(k+1) → lower is better; prefer for GOODNESS OF FIT

SSE = sum of squared errors (residuals²) from the regression
n = observations | k = independent variables | ln = natural log
2(k+1) / ln(n)(k+1) = complexity penalty (BIC penalizes harder → more parsimonious model)
Adding a variable that increases R² but decreases adjusted R² = the variable adds noise, not signal. Use AIC/BIC to compare across different variable combinations.

F-Test (Overall Significance)

Joint F-Test
H₀: all slope coefficients = 0 (model has no explanatory power)
F = [R²/k] / [(1−R²)/(n−k−1)] ~ F(k, n−k−1)
Reject H₀ (all β=0) if F > critical F value from F-table

k = degrees of freedom numerator (# independent variables)
n−k−1 = degrees of freedom denominator

Restricted vs Unrestricted (joint test on a SUBSET of variables):
F = [(SSE_R − SSE_U)/q] / [SSE_U/(n−k−1)]
SSE_R = sum of squared errors from restricted model (variables dropped)
SSE_U = sum of squared errors from unrestricted model (all variables included)
q = number of restrictions = number of variables dropped
High R² + significant F-stat + insignificant individual t-stats = multicollinearity. The model fits, but can't attribute credit to individual variables.

Forecasting

Plug estimated values of X into the regression equation to get a predicted Ŷ. Confidence interval around the forecast widens as X values move further from their means (wider uncertainty at extremes). Out-of-sample forecast reliability depends on the model being correctly specified and stable.

Model Misspecification

▶

One Rule Covers All Three

Heteroskedasticity, serial correlation, and multicollinearity all bias standard errors but NOT the regression coefficients. Coefficients remain unbiased — only inference is unreliable.

ViolationBiases β̂?Biases SE?t-stat effectTestFix
Conditional HeteroskedasticityNoYesUnreliableBreusch-PaganWhite's robust SE
Positive Serial CorrelationNoUnderstatedToo high → Type I ↑Durbin-WatsonNewey-West/Hansen
MulticollinearityNoOverstatedToo low → Type II ↑VIFDrop a variable

Tests

Breusch-Pagan (Heteroskedasticity)
Step: regress squared residuals (ε̂²) on all original X variables
BP = n × R²_auxiliary ~ χ²(k)
n = number of observations | R²_auxiliary = R² from that auxiliary regression
k = number of X variables | χ²_critical from chi-squared table at significance level
Reject H₀ (homoskedasticity) if BP > χ²_critical → heteroskedasticity present
Durbin-Watson (Serial Correlation)
DW = 2(1 − r̂₁) where r̂₁ = first-order serial correlation of residuals
DW ≈ 2 → no serial correlation
DW → 0 → strong positive serial correlation (errors move together)
DW → 4 → strong negative serial correlation (errors alternate sign)
d_L, d_U = lower and upper critical values from DW table
DW < d_L → reject H₀ | DW > d_U → fail to reject | d_L < DW < d_U → inconclusive
⚠ Do NOT use DW when lagged Y appears as a regressor (AR models)
VIF (Multicollinearity)
VIF_j = 1 / (1 − R²_j)
R²_j = R² obtained by regressing variable X_j on all other independent variables
Measures how much the variance of β̂_j is inflated due to correlation with other X's
VIF = 1: no multicollinearity | VIF 1–5: acceptable | VIF 5–10: moderate | VIF >10: severe

Dummy Variables

For n categories use n−1 dummies. Including all n = perfect multicollinearity (dummy variable trap).
Intercept Dummy / Slope Dummy
Intercept: Y = b₀ + d₀D + b₁X → D=1 shifts the intercept by d₀ (parallel line)
Slope (interaction): Y = b₀ + b₁X + d₁(D·X) → D=1 changes slope to b₁+d₁

Logistic Regression

Logistic Model (binary Y)
ln[p/(1−p)] = b₀ + b₁X₁ + ... + bₖXₖ (log-odds / logit form)
p = probability of Y=1 (the event occurring)
p/(1−p) = odds ratio | ln[p/(1−p)] = log-odds (logit)
b₀ = intercept | bⱼ = change in log-odds per 1-unit ↑ in Xⱼ

Solving for probability: p = 1 / (1 + e^{−(b₀+b₁X₁+...+bₖXₖ)})
e = Euler's number (≈2.718) | e^(bⱼ) = odds ratio for Xⱼ
Estimated via Maximum Likelihood Estimation (MLE), not OLS
Tested via z-test (large n) or likelihood ratio (LR) test for overall model

Time-Series Analysis

▶

Trend Models

Linear vs Log-Linear
Linear: y_t = b₀ + b₁t + ε_t → constant AMOUNT growth per period
Log-linear: ln(y_t) = b₀ + b₁t + ε_t → constant RATE (exponential) growth
Switch to log-linear if linear trend residuals show serial correlation

AR Models

AR(1) — Mean Reversion
x_t = b₀ + b₁x_{t-1} + ε_t
x_t = value of the series at time t
b₀ = intercept | b₁ = slope on lagged value (must be |b₁| < 1 for stationarity)
ε_t = error term (white noise, uncorrelated across time)
Mean-reverting level = b₀/(1−b₁) ← the long-run average the series gravitates toward
Example b₀=0.4, b₁=0.8 → mean-reverting level = 0.4/0.2 = 2.0
x_t=5 (above mean) → x_{t+1} = 0.4+0.8(5) = 4.4 → moving toward 2.0

Unit Root & Dickey-Fuller

DF Test
Transform: Δx_t = γ·x_{t-1} + ε_t where γ = b₁ − 1
Δx_t = x_t − x_{t-1} = first difference of the series
γ = b₁ − 1 (the coefficient being tested)
H₀: γ = 0 (b₁ = 1 → unit root → nonstationary)
H₁: γ < 0 (b₁ < 1 → stationary)
Reject H₀ if DF test stat is MORE NEGATIVE than DF critical value (≈−2.86 at 5%)
DF critical values are much more negative than standard t-critical values
If unit root found → first-difference: use Δx_t = x_t − x_{t-1} as new dependent variable
Cointegration: two nonstationary series where a linear combination is stationary → regression is valid. Test: DF test on the residuals of the regression.

ARCH & Seasonality

ARCH(1) | Seasonal AR
ARCH: σ²_t = a₀ + a₁ε²_{t-1}. Detect: regress ε̂² on lagged ε̂². Fix: GARCH
Seasonality: add seasonal lag, e.g., x_{t-4} for quarterly data

Currency Exchange Rates

▶

FX Basics

Quote = P/B (Price / Base). USD/EUR = 1.35 → 1 EUR costs 1.35 USD. Base in the denominator. Bid = dealer buys base (lower). Ask = dealer sells base (higher).

Cross Rate | Forward Rate (CIP)
A/C = (A/B) × (B/C) — B cancels. Bid of cross = bid₁ × bid₂. Ask = ask₁ × ask₂.
Inverting: new bid = 1/old ask | new ask = 1/old bid

F_{P/B} = S_{P/B} × [1 + i_P(d/360)] / [1 + i_B(d/360)]
F_{P/B} = forward exchange rate (price currency per base currency)
S_{P/B} = spot exchange rate today
i_P = interest rate in the price currency (e.g., USD if quote is USD/EUR)
i_B = interest rate in the base currency (e.g., EUR if quote is USD/EUR)
d = days in the forward contract (e.g., 90, 180, 360)
i_P > i_B → F > S (base at forward discount — high-rate currency weakens in forward market)

Triangular Arbitrage

DBi G IO (dealer bid > interbank offer): buy from interbank, sell to dealer.
DO L IB (dealer offer < interbank bid): buy from dealer, sell to interbank.

Number line: dealer quote entirely LEFT of interbank range → DO L IB (buy from dealer). Entirely RIGHT → DBi G IO (buy from interbank). Any overlap = no arbitrage.

Five Parity Conditions

ConditionFormulaHolds?
CIP (Covered IRP)F/S = (1+i_P)/(1+i_B) — hedge with forward contractMust hold (arbitrage enforced)
UIP (Uncovered IRP)%ΔS_expected = i_P − i_B — no hedgeFails short-term → carry trade profits
Relative PPP%ΔS ≈ π_P − π_B (inflation differential)Long-run only
Fisher Effecti_nominal = r_real + πGenerally holds
Real IRPr_real_P = r_real_B (real rates equalize globally)Long-run only
Carry Trade Intuition Borrow JPY at 0.5%, invest AUD at 5%. UIP says AUD should depreciate 4.5% → net = 0%.
In practice, AUD often doesn't depreciate enough → carry trade pockets the differential.
Risk: in risk-off environments carry unwinds violently (AUD crashes fast).

FX Determination Models

  • Mundell-Fleming: Expansionary monetary policy (floating rate) → depreciation. Expansionary fiscal (fixed rate) → appreciation.
  • Monetary Models: ↑ money supply → ↑ domestic prices → ↓ exchange rate (consistent with PPP)
  • Portfolio Balance: Persistent current account deficits → investors demand risk premium → gradual depreciation

Currency Crisis Warning Signs

  • Large current account deficit
  • Declining foreign exchange reserves
  • Currency pegged at overvalued rate relative to PPP
  • High domestic inflation relative to peg partner
  • Rapid domestic credit expansion

Economic Growth & the Investment Decision

▶

Why Growth Matters

Stock Price Identity
P = GDP × (E/GDP) × (P/E)
Long-run: E/GDP and P/E mean-revert → stock prices track GDP
Long-run real EPS growth cannot persistently exceed potential GDP growth

Cobb-Douglas & Growth Accounting

Production Function
Y = TFP × K^α × L^(1−α)
Y = real GDP (total economic output)
TFP = Total Factor Productivity = technology/efficiency (the "Solow residual")
K = capital stock (machines, buildings, infrastructure)
L = labor input (number of workers or hours)
α = capital's share of output (typically ≈ 1/3); 1−α = labor's share (≈ 2/3)

Growth accounting: ΔY/Y = ΔTFP/TFP + α(ΔK/K) + (1−α)(ΔL/L)
TFP growth is the residual — what's left after accounting for capital and labor growth

Growth Models

ModelKey ClaimLong-Run Driver
ClassicalTech → population ↑ → income falls to subsistence (Malthusian)None
Neoclassical (Solow)Diminishing returns to capital → only TFP drives long-run per capita growthExogenous TFP
EndogenousNo diminishing returns to broad capital; R&D and knowledge spill overPolicy, R&D, human capital
Key Intuition — Neoclassical Steady State Each additional machine adds less output than the last (diminishing returns).
Eventually new investment just replaces depreciation → no net growth from capital alone.
Only TFP growth permanently raises the per capita growth rate.

Cost of Capital: Advanced Topics

▶

WACC

WACC Formula
WACC = w_d·r_d(1−t) + w_p·r_p + w_e·r_e
Weights = TARGET capital structure (not current book or market values)
r_d = YTM on existing debt | r_e = CAPM, DDM, or bond yield + premium

Beta: Unlevering & Relevering

Beta Adjustments
Unlever: β_U = β_L / [1 + (1−t)(D/E)]
Re-lever: β_L = β_U × [1 + (1−t)(D/E)]
Example Comparable: β_L=1.2, D/E=0.5, t=30% → β_U = 1.2/1.35 = 0.889
Your target D/E=1.0 → β_L = 0.889 × 1.7 = 1.51 → higher leverage → higher required return

Factors Affecting WACC

  • Top-down: Capital availability, market conditions (interest rates, credit spreads), legal system, tax rates
  • Bottom-up: Revenue/earnings volatility, asset tangibility, leverage, ESG risk, security features (callability, putability)
  • Callable debt → higher yield demanded (issuer benefit). Putable debt → lower yield (investor benefit).

Discounted Dividend Valuation

▶

Gordon Growth Model & Justified Multiples

GGM | Justified Multiples
V₀ = D₁/(r−g) = D₀(1+g)/(r−g)
V₀ = intrinsic value of stock today
D₀ = current dividend just paid | D₁ = next expected dividend (= D₀ × (1+g))
r = required return on equity (cost of equity)
g = constant perpetual dividend growth rate [must have r > g]
b = retention ratio = fraction of earnings retained (1 − payout ratio)
g (sustainable) = ROE × b (earnings reinvested × return earned on reinvestment)

Justified P/E = (1−b)/(r−g) [1−b = payout ratio]
Justified P/B = (ROE−g)/(r−g)

Free Cash Flow Valuation

▶

FCFF & FCFE

FCF Formulas — Three Ways to Compute FCFF
FCFF (Free Cash Flow to the Firm) — available to ALL capital providers (debt + equity)
From NI: FCFF = NI + D&A + Interest(1−t) − ΔWC − CapEx
From EBIT: FCFF = EBIT(1−t) + D&A − ΔWC − CapEx
From CFO: FCFF = CFO + Interest(1−t) − CapEx

FCFE (Free Cash Flow to Equity) — available to EQUITY holders only
FCFE = FCFF − Interest(1−t) + Net new borrowing
FCFE = CFO − CapEx + Net new borrowing

NI = net income | D&A = depreciation and amortization (non-cash, add back)
Interest(1−t) = after-tax interest expense | t = tax rate
ΔWC = change in net working capital (↑WC = cash use → subtract)
CapEx = capital expenditures (net of asset sale proceeds)
CFO = cash flow from operations (from statement of cash flows)

FCFF → discount at WACC → firm value → subtract market value of debt → equity value
FCFE → discount at r_e → equity value directly
Most common error: starting from NI to FCFF and forgetting to ADD BACK Interest(1−t). FCFF is pre-debt; you must add interest back. Going from FCFF → FCFE: SUBTRACT interest(1−t) and ADD net new borrowing.

Key Adjustments in FCF Models

  • ΔWC: Increase in working capital = cash use (subtract). Decrease = cash source (add). Non-cash WC only (exclude cash and short-term debt).
  • CapEx: Net CapEx = gross CapEx − proceeds from asset sales
  • Non-cash charges: D&A, impairments, restructuring charges — add back to NI
  • FCFF vs FCFE: Use FCFF when leverage is changing or unstable — avoids distortions from financing. Use FCFE for stable leverage.
  • Negative FCF: Common for high-growth companies. Not a problem if funded by debt/equity and invested in positive-NPV projects. Terminal value drives most of the value.

FCF Model Application

Multistage FCF Valuation
Phase 1: Forecast FCFF/FCFE explicitly (typically 5–10 years)
Phase 2: Terminal value = FCFF_{n+1} / (WACC − g) or FCFE_{n+1} / (r_e − g)
V_firm = Σ[FCFF_t/(1+WACC)^t] + TV/(1+WACC)^n
V_equity = V_firm − Market value of debt

Residual Income Valuation

▶

Residual Income (Economic Profit)

RI Formula
RI_t = EPS_t − r × B_{t-1} = (ROE_t − r) × B_{t-1}
RI_t = residual income in period t (economic profit above the equity cost hurdle)
EPS_t = earnings per share in period t
r = required return on equity (cost of equity = the "hurdle rate")
B_{t-1} = book value per share at beginning of period t (end of prior period)
ROE_t = return on equity in period t = EPS_t / B_{t-1}

V₀ = B₀ + Σ[RI_t / (1+r)^t]
V₀ = intrinsic value today | B₀ = current book value per share
Single-stage: V₀ = B₀ + [(ROE−r)/(r−g)] × B₀
g = constant growth rate of book value and RI
Worked Example B₀=$20, ROE=18%, r=12%, g=5%
V₀ = $20 + [(0.18−0.12)/(0.12−0.05)] × $20 = $20 + 0.857 × $20 = $37.14
Premium to book = $17.14 because ROE > r — value is being created above the cost of equity every period

Key Insights & Persistence

  • If ROE = r perpetually → V₀ = B₀ (trades at book value, no premium)
  • If ROE > r → V₀ > B₀ (premium to book)
  • If ROE < r → V₀ < B₀ (discount to book — value-destroying company)
  • Persistence factor (ω): How long above-normal RI persists. ω=1 → RI continues indefinitely. ω=0 → RI falls to zero next period. Most realistic: ω between 0 and 1.
RI vs Other Models
Advantage: anchors valuation in observable book value; useful when FCF negative
Advantage: most value attributed to existing assets (less sensitive to terminal value assumptions)
Disadvantage: relies on accounting book value; goodwill, intangibles distort B₀
Disadvantage: requires clean surplus accounting (all items flow through retained earnings)

(1+z_B)^B = (1+z_A)^A × (1+f_{A,B-A})^{B-A}

If spot curve upward sloping → forward curve lies ABOVE spot curve
If spot curve flat → forward rates equal spot rates

If spot curve upward sloping → forward curve lies ABOVE spot curve
If spot curve flat → forwards = spots

OAS is the apples-to-apples spread for comparison across bonds with different embedded options. Higher OAS vs peers → cheap relative to model.

Lattice of interest rates where each node represents a possible future short rate. Used to value bonds with embedded options. Work backward: find value at each node using risk-neutral probabilities (50/50 up/down) and discount. For callable bonds, cap node value at call price.

Credit Analysis Models

▶

Credit Spread & Expected Loss

Credit Spread Framework
Credit spread ≈ POD × LGD
POD = Probability of Default over the period
LGD = Loss Given Default = amount lost if default occurs = 1 − Recovery rate
Recovery rate = fraction of face value recovered after default (typically 40% for senior unsecured)
Hazard rate (λ) = instantaneous conditional POD given no prior default has occurred
Unconditional POD_t = λ × (1 − cumulative POD through t−1)

Expected loss = POD × LGD × Notional
Risk-neutral POD (implied from market spreads, includes risk premium) ≠ actual historical default rates

The Four Cs of Credit Analysis

  • Capacity: Ability to repay — leverage ratios (D/EBITDA, D/E), coverage ratios (EBIT/interest), free cash flow generation, business cyclicality
  • Collateral: Asset quality; priority in capital structure (senior secured > senior unsecured > subordinated > equity)
  • Covenants: Contractual protections for creditors. Maintenance covenants: must be met continuously (common in bank loans). Incurrence covenants: triggered only by specific actions (common in high-yield bonds).
  • Character: Management track record, strategic credibility, governance, history of treating creditors

Structural vs Reduced-Form Credit Models

Model TypeKey AssumptionDefault occurs when...
Structural (Merton)Default when asset value falls below debt obligation (option-based)V_assets < V_debt at maturity. Equity = call option on firm assets.
Reduced-FormDefault is exogenous, modeled as a Poisson process with hazard rateRandom arrival governed by calibrated hazard rate from market data
Structural models give intuition (equity as call option, credit as put option on firm assets). Reduced-form models are more practical for pricing (calibrate to market spreads directly).

BSM Formulas (with dividends γ)

X = strike price (exercise price)
r = continuously compounded risk-free interest rate (annualized)

γ = continuous carry benefit yield (δ = dividend yield for stocks; r_f = foreign rate for FX)
e = Euler's number (≈2.718) | e^(−rT) = present value discount factor
N(x) = standard normal CDF = probability that a standard normal variable is ≤ x

    Value Added Through Active Management

    Active Return & Active Weights
    R_A = R_P − R_B (active return = value added)

    R_A = Σ(Δwᵢ × RAᵢ) where RAᵢ = Rᵢ − R_B

    IR | Sharpe | Grinold-Kahn
    IR = E[R_A] / σ(R_A) (active return / tracking error)
    IR ≥ 0.5: good | IR ≥ 1.0: exceptional

    Sharpe ratio: SR = (R_P − R_f) / σ_P
    Grinold-Kahn: SR_P² = SR_B² + IR²
    → A manager with IR=0.5 improves portfolio SR even when benchmark SR is already good

    Fundamental Law of Active Management

    Fundamental Law
    Basic: IR = IC × √BR
    Full (with constraints): IR = TC × IC × √BR

    IR = Information Ratio = E[active return] / σ(active return) = active reward-to-risk
    IC = Information Coefficient = correlation between forecasted and actual active returns
    IC = 0 → no skill (random) | IC = 1 → perfect foresight | typical good manager: 0.05–0.15
    BR = Breadth = number of INDEPENDENT investment decisions made per year
    Must be INDEPENDENT — correlated bets count as fewer than their face number
    TC = Transfer Coefficient = correlation between alpha forecasts and actual active weights
    TC = 1 if unconstrained (all forecasts fully translated into positions)
    TC < 1 if long-only constraint, position limits, or other portfolio construction constraints
    Breadth vs Skill Tradeoff Manager A: IC=0.10, BR=100 → IR = 0.10×10 = 1.0
    Manager B: IC=0.20, BR=25 → IR = 0.20×5 = 1.0
    Equal IR — A makes more bets with less skill; B makes fewer bets but is sharper.
    Key: doubling independent bets has same IR effect as doubling IC. Breadth matters.

    Active vs Passive; Closet Indexing

    • Closet indexer: Small active weights → low tracking error → low IR despite active fees. Client paying for alpha but getting near-index returns.
    • Optimal active portfolio: Add securities with positive expected active return relative to their active risk contribution

    Factor Model Basics

    Return Decomposition
    R_i = α_i + β_{i,1}F₁ + β_{i,2}F₂ + ... + β_{i,k}Fₖ + ε_i

    Active return decomposition:
    R_A = Σ(β_P,j − β_B,j)·F_j + ε_A
    = active factor tilts × factor returns + pure security selection

    Types of Factor Models

    TypeFactorsProsCons

    Uses of Multifactor Models

    • Risk decomposition: How much of portfolio variance comes from factor exposures vs idiosyncratic risk
    • Performance attribution: Which factor tilts drove returns vs security selection
    • Portfolio construction: Constrain factor exposures; target specific factor tilts
    • Closet indexing detection: Low active factor exposures + low ε_A → manager isn't really adding alpha
    Tracking error comes from BOTH active factor tilts AND security selection. A manager can have zero tracking error by perfectly matching benchmark factor exposures — this is a closet indexer charging active fees for passive exposure.

    Measuring & Managing Market Risk

    ▶

    VaR

    VaR & Normal Distribution Levels
    VaR = minimum loss expected to be exceeded with a given probability over a given horizon
    "5% daily VaR = $2M" → on 5% of days (1 in 20), the loss WILL BE AT LEAST $2M
    Equivalently: 95% confidence that daily loss will NOT exceed $2M

    Normal distribution z-scores (memorize):
    1% VaR → 2.33σ below mean (99% confidence)
    5% VaR → 1.65σ below mean (95% confidence)
    16% VaR → 1.00σ below mean (84% confidence)

    σ = standard deviation of portfolio returns
    Scaling across time (assumes i.i.d. returns):
    σ_T-day = σ_1-day × √T
    VaR_monthly ≈ VaR_daily × √21 (21 trading days per month)
    MethodAssumptionProCon
    ParametricNormal distributionSimple, fastBad for fat tails; options
    Historical SimulationPast distributions repeatCaptures fat tails; no dist assumedLimited by history
    Monte CarloSpecified distributionsFlexible; handles complexityComputationally intensive
    VaR = MINIMUM loss at threshold — says nothing about losses beyond it. CVaR (Expected Shortfall) = expected loss given you're in the tail. More informative for risk management.

    Backtesting & Simulation

    ▶

    Backtesting Biases — All Four Must Know

    BiasWhat It IsEffect
    Survivorship biasOnly includes securities/funds that survived to end of periodOverstates performance — dead companies, bankrupt funds excluded
    Look-ahead biasUses data not actually available at time of investment decisionOverstates performance — impossible to replicate live
    Data snooping / overfittingTesting many strategies on same dataset until one "works" by chanceStrategy unlikely to work out-of-sample; p-hacking
    Transaction cost omissionIgnores bid-ask spreads, commissions, market impact, slippageSignificantly overstates performance for high-turnover strategies
    Out-of-sample testing is the most reliable defense against data snooping. Walk-forward testing (train on past, test on future) is more realistic than random cross-validation for time-series data.

    Simulation Methods

    MethodWhat It DoesBest For
    Historical simulationReplay actual historical return sequencesCapturing real correlation structures; stress testing using known crises
    Monte Carlo simulationGenerate thousands of random scenarios from specified distributionsComplex portfolios; instruments with embedded options; path-dependent payoffs
    Sensitivity analysisChange one assumption at a time; observe impactIdentifying which inputs drive the most uncertainty
    Scenario analysisDefine specific multi-variable stress scenarios (e.g., 2008 replay, rate shock)Regulatory stress testing; tail risk assessment

    Key Limitations

    • Backtests cannot validate strategies that exploit structural changes not present in history
    • More historical data reduces sampling error but may include irrelevant regimes
    • Number of observations needed grows with the number of parameters tested (degrees of freedom)
    • Strategy capacity: a strategy that worked historically may not scale to large AUM due to market impact

    Guidance for Standards I–VII — Complete

    ▶

    Standard I — Professionalism

    I(A) Knowledge of the Law

    Follow the STRICTER of applicable law or the Code and Standards. If conflict between two laws, follow the stricter one. If a violation is occurring: attempt to stop it → escalate to supervisor/compliance → seek independent legal counsel → dissociate from the activity → resign in extreme cases. CFA Institute does NOT require you to report violations to regulators unless required by law.

    I(B) Independence & Objectivity

    Do not accept gifts, benefits, or compensation that could compromise independence. Key scenarios:

    • Buy-side client pressure: Must not issue favorable recommendations just because a large client holds the stock
    • Investment banking relationships: IB division cannot pressure research analysts to issue favorable reports on IB clients
    • Issuer-paid research: Negotiate a flat fee independent of conclusion; disclose the compensation arrangement
    • Travel funding: Use commercial transport; if unavailable, accept only modest travel. Never extravagant issuer-funded travel.
    • Credit rating agencies: Must remain objective even if issuer provides incentives for favorable rating
    Modest gifts are OK. Gifts that "reasonably could be expected to compromise" objectivity are NOT. The line is whether it could influence your professional judgment — not whether it actually did.

    I(C) Misrepresentation

    Must not knowingly misrepresent qualifications, investment process, performance, or facts. Key areas:

    • Performance reporting: Don't cherry-pick periods; include terminated accounts in composite performance
    • Plagiarism: Cannot use others' work without attribution — including charts, research, or unnamed "leading analysts"
    • Social media: Same standards apply online as in traditional communication
    • Work for employer: Cannot claim work done by someone else as your own; but employer owns work product, not the departed employee

    I(D) Misconduct

    No dishonesty, fraud, or deceit. Personal bankruptcy alone ≠ misconduct; but if circumstances involve fraud, it is. Alcohol use during business hours that impairs judgment = potential violation.

    I(E) Competence

    Must maintain the knowledge, skills, and abilities required for your specific professional role. Must update skills as responsibilities change. Participate in continuing education, certifications, training.

    Standard II — Integrity of Capital Markets

    II(A) Material Nonpublic Information (MNPI)

    Material: Information that, if disclosed, would affect the price of a security or that investors would want to know before making investment decisions (earnings surprises, M&A, new product approvals, management changes, legal issues).
    Nonpublic: Not yet disseminated to the public through official channels.

    • Mosaic Theory: Combining public information AND nonmaterial nonpublic information to reach a conclusion is legal. No single piece was MNPI. The nonpublic conclusion is permitted.
    • Firewall: Information barrier between departments (e.g., IB vs research). Route interdepartmental comms through compliance. Physical separation of departments.
    • Social media: Closed groups on social media — treat as nonpublic unless same info available through other public sources
    Mosaic Theory Example Analyst reads SEC filing [public] + industry reports [public] + calls retailers who confirm slow orders [nonmaterial nonpublic] → concludes company will miss.
    Legal: each piece was public or nonmaterial. The CONCLUSION is nonpublic but that's the analyst's value-add.
    MNPI would be: CFO privately says "we'll miss by 30%" before public announcement.

    II(B) Market Manipulation

    Cannot engage in practices that distort prices or artificially inflate volume with intent to mislead.

    • Information-based: Spreading false rumors through blogs, media to inflate/deflate prices
    • Transaction-based: Wash trades (firm buys and sells simultaneously to create fake volume), painting the tape

    Standard III — Duties to Clients

    III(A) Loyalty, Prudence & Care

    Fiduciary duty: client interests first, then employer, then personal. The actual "client" may differ from who hired you (pension fund: client = beneficiaries, not the pension administrator).

    • Soft dollars: Using client brokerage to purchase research services is acceptable ONLY if it benefits clients. Using for office equipment, personal travel = violation.
    • Proxy voting: Must vote proxies in the best interest of the client/shareholder — not company management. Disclose proxy voting policies.
    • Best execution: Seek best execution for client trades. Directed brokerage arrangements must still be disclosed and must be in clients' interest.

    III(B) Fair Dealing

    Disseminate recommendations fairly and simultaneously to all clients. Different service LEVELS are OK (premium clients get more resources). Different INFORMATION is not.

    • Limit people who know about pending recommendations before dissemination
    • Shorten time between investment decision and dissemination
    • IPO allocations: pro-rata to all suitable accounts; don't favor certain clients
    • Block trades: all clients get same execution price and commission
    • Oversubscribed IPOs: distribute pro-rata; managers must not front-run allocation for their own accounts

    III(C) Suitability

    Know your client before recommending. Gather return objectives, risk tolerance, time horizon, liquidity needs, tax situation, legal constraints. Maintain an IPS; update at least annually.

    • Judge suitability in context of the ENTIRE portfolio — a speculative security may be appropriate as a small allocation even if risky in isolation
    • For institutional clients (pension funds, endowments): evaluate suitability for the PORTFOLIO mandate, not the individual security
    • If client insists on unsuitable trade: document, educate, update IPS if needed. If client refuses to update IPS, consider ending advisory relationship.

    III(D) Performance Presentation

    • Performance must be fair, accurate, and complete — no cherry-picking periods or portfolios
    • Include terminated accounts in performance history (survivorship bias)
    • Cannot state that past performance guarantees future results
    • If not GIPS-compliant: use weighted composite of similar portfolios; disclose benchmark selection rationale; maintain supporting records

    III(E) Preservation of Confidentiality

    Keep client information confidential — including former and prospective clients. Exceptions: (1) client's own illegal activities, (2) required by law, (3) client permits disclosure. Must cooperate with CFA Institute PCP investigations if permissible under law.

    Standard IV — Duties to Employers

    IV(A) Loyalty to Employer

    • Cannot deprive employer of skills, divulge confidential information, or harm employer
    • Independent practice: Must get written consent from employer before starting side work; disclose nature, duration, compensation
    • Leaving employer: During notice period: continue acting in employer's best interest; don't reveal trade secrets; don't solicit clients to move with you. After leaving: can use skills and general knowledge; CANNOT take records, models, or client lists in paper or electronic format.
    • Whistleblowing: Acceptable if intent is to protect capital market integrity or client interests. NOT acceptable if for personal gain.

    IV(B) Additional Compensation Arrangements

    Cannot accept compensation that conflicts with employer's interest without written consent from all parties. Must immediately disclose in writing to supervisor/compliance officer. Gifts of significant value require disclosure and approval. Token gifts (pens, calendars) need not be reported.

    IV(C) Responsibilities of Supervisors

    • Must make reasonable efforts to ensure those under supervision comply with applicable laws and the Code and Standards
    • If compliance procedures are inadequate → must bring to senior management's attention; if still inadequate → decline supervisory responsibility
    • When violation is detected: immediately report, initiate investigation — NOT sufficient to just warn the employee
    • Delegating supervisory duties does NOT absolve supervisor of responsibility

    Standard V — Investment Analysis, Recommendations & Actions

    V(A) Diligence & Reasonable Basis

    • Must have reasonable and adequate basis for all recommendations
    • When using third-party research: must independently evaluate it — cannot just rely on rating agencies or external managers without your own assessment
    • Group research: if you disagree but believe consensus has a reasonable basis and is objective, you need not dissociate or remove your name
    • Quantitative models: understand data, parameters, assumptions, and limitations; test under multiple scenarios including adverse conditions

    V(B) Communication with Clients

    • Disclose the nature and cost of services provided
    • Explain the investment process (not just the conclusion) including limitations and risks
    • Promptly disclose any changes to the investment process that materially affect clients
    • Distinguish facts from opinions in reports — clearly label forward-looking views as opinions
    • Applies to all communication channels: written reports, social media, phone calls, emails

    V(C) Record Retention

    • Maintain records supporting investment analyses, recommendations, and client communications
    • CFA Institute recommends retaining records for at least 7 years
    • Records are property of the FIRM — cannot take them when leaving without employer consent
    • Social media posts are records and must be retained per firm policy

    Standard VI — Conflicts of Interest

    VI(A) Avoid or Disclose Conflicts

    • Best practice: avoid the conflict. When unavoidable: disclose in plain language, prominently and effectively.
    • Must disclose: personal stock ownership in securities you recommend, IB relationships, compensation arrangements, board memberships
    • Cross-departmental conflicts: if IB pressures research to issue favorable report → note IB relationship in report
    • If firm doesn't permit disclosure: dissociate from the activity; document the request

    VI(B) Priority of Transactions

    Order: Clients → Employer → Personal (including beneficial ownership in family accounts)

    • Front-running = always a violation regardless of trade size
    • May trade in personal accounts only after clients and employer have had adequate opportunity to act
    • Personal trading policies: pre-clearance, blackout periods, reporting requirements all help ensure compliance

    VI(C) Referral Fees

    Must disclose any referral fee arrangements to clients and prospective clients before accepting engagement. Must also disclose to employer. Clients can then assess whether the recommendation is biased by the referral arrangement.

    Standard VII — CFA Member/Candidate Responsibilities

    VII(A) Conduct in CFA Programs

    • Cannot violate exam security policies — discussing exam questions (even after the session, with candidates in other time zones who haven't yet sat) = violation
    • Cannot falsify reporting of hours on CFA exam applications
    • Violations can result in suspension or ban from the CFA Program

    VII(B) CFA Designation Reference

    • Cannot use "CFA" as a noun (e.g., "I am a CFA" is wrong → "I am a CFA charterholder" is correct)
    • Cannot refer to having passed Level I or II as equivalent to being a charterholder
    • Cannot use the CFA designation if your membership is suspended or revoked
    • Correct: "CFA charterholder," "passed Level II of the CFA examination" (not "CFA Level II")

    Soft Dollars Deep Dive — Commonly Tested

    Soft Dollar Rules
    Acceptable: directing client brokerage to brokers who provide research that benefits clients
    NOT acceptable: using client brokerage for:
    · Office equipment, furniture, rent, salaries
    · Personal travel, entertainment
    · Any benefit that does not directly benefit clients

    Must ensure: brokerage commissions paid are reasonable relative to research received
    Must disclose: soft dollar arrangements to clients

    GIPS — Complete

    • Voluntary, firm-wide standards for presenting investment performance history to prospective clients
    • No cherry-picking: All actual, fee-paying, discretionary portfolios must be included in at least one composite
    • Performance history: Must present 5 years of GIPS-compliant performance (or since inception if <5 years), then build to 10 years
    • Composites: Group portfolios with similar strategies/objectives. Cannot create composites retroactively to show favorable performance.
    • Required: Trade date accounting, accrual accounting for fixed income, time-weighted return (TWR)
    • Terminated accounts: Must include in composite performance through the last full measurement period they were active
    • Verification: Voluntary; performed by independent third party; verifies firm-wide GIPS compliance (not individual composite accuracy)
    • Claiming compliance: Must be firm-wide; individual divisions cannot claim GIPS compliance independently
    • Prospective clients: Must receive a GIPS-compliant presentation on request; cannot show non-GIPS to some and GIPS to others
    GIPS compliance = firm-wide or nothing. You cannot say "our equity division is GIPS compliant" if the firm as a whole is not. Verification ≠ audit of individual composites — it verifies the overall process.
